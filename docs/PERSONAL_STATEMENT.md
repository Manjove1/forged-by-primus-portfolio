# Personal Statement

I design structured AI systems that keep state coherent, enforce meaningful constraints, and recover cleanly when context degrades. Open-ended AI roleplay tools often lose mechanical consistency, skip multi-step processes, and treat every action as equally valid. When consequence disappears, player decisions lose weight. I wanted systems that preserved consequence without flattening narrative quality.

*Forged by Primus* is the practical result of that work: a modular AI Dungeon Master framework built around an orchestrator pattern, self-healing state tracking, sequential character creation gates, and recovery protocols designed for platforms that aggressively summarize context. The project is less a single prompt than a coordinated set of systems that stay reliable across long sessions.

My development process is deliberate. I establish foundations first, then iterate and backfill as new requirements appear. That brick-by-brick approach fits interconnected system design well, especially when each addition has to earn its place against token cost, maintainability, and playability. Working with AI tools that can match that pace, pressure-test assumptions, and refine structure in real time has become a core part of how I build.

The project began from a long-standing interest in how games create meaningful stories and a deep familiarity with the Transformers setting. The setting provided creative drive. The harder technical problems — state reliability, process enforcement, multi-body systems, and recovery from context loss — became the real center of the work. What I care about most is building transferable structures: systems that let an AI do specific things well, with consistency and recoverability, across settings rather than only inside one franchise world.
